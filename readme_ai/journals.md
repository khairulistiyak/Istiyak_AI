"আমার প্রজেক্টের ফ্রন্টএন্ডে /admin রাউটে একটি অ্যাডভান্সড, এন্টারপ্রাইজ-গ্রেড পার্সোনাল ড্যাশবোর্ড তৈরি করো। এটি Next.js, React, TypeScript, এবং Tailwind CSS ব্যবহার করে বানাতে হবে। নিচের সবগুলো রিকোয়ারমেন্ট একদম নিখুঁতভাবে ইমপ্লিমেন্ট করো:

১. প্রিমিয়াম UI/UX (গ্লাসমরফিজম এবং ডার্ক মোড):

ড্যাশবোর্ডটি হবে একদম মডার্ন। Tailwind ব্যবহার করে Glassmorphism ইফেক্ট (backdrop-blur, semi-transparent backgrounds) যুক্ত করো।

একটি সুন্দর (sleek) সাইডবার থাকবে। UI-তে প্রিমিয়াম ফিল আনার জন্য স্মুথ অ্যানিমেশন এবং ট্রানজিশন ব্যবহার করবে।

২. অ্যাডভান্সড পোস্ট বক্স (মার্কডাউন এবং ট্যাগিং):

সাধারণ ইনপুট ফিল্ডের (textarea) বদলে একটি মার্কডাউন এডিটর (Markdown Editor, যেমন react-simplemde-editor বা react-quill) ব্যবহার করো, যাতে আমি কোড স্নিপেট বা স্টাইল করা টেক্সট লিখতে পারি এবং সেটি ডাটাবেসে সেভ হয়।

এডিটরের নিচের দিকে একটি 'Tag/Category Selector' থাকবে। আমি যেন খুব সহজেই বিভিন্ন ক্যাটাগরি (যেমন: Technology, Hardware, Personal, Brand) সিলেক্ট করতে পারি।

৩. ডেটাবেস স্কিমা এবং API রাউট (Mongoose/MongoDB):

creator_journals কালেকশনের স্কিমা হবে: { content: String, tags: [String], isActive: Boolean (default: true), createdAt: Date }।

/api/journals রাউটে ডেটা সেভ (POST), ডেটা ফেচ (GET), এবং স্ট্যাটাস আপডেটের জন্য (PATCH) এপিআই তৈরি করো।

৪. ফিড এবং অ্যাক্টিভেশন টগল:

ড্যাশবোর্ড থেকে পোস্ট করার পর নিচের ফিডে পোস্টগুলো লিস্ট আকারে দেখাবে।

প্রতিটি পোস্টের পাশে একটি 'Toggle Switch' (Active/Inactive) থাকবে। টগল অফ করলে PATCH রিকোয়েস্টের মাধ্যমে ডেটাবেসে সেটি isActive: false হয়ে যাবে এবং এআই সেটি আর পড়বে না।

৫. RAG (Retrieval-Augmented Generation) রেডি আর্কিটেকচার:

aiService.ts বা ব্যাকএন্ডে getRelevantContext(query) নামের একটি মডিউলার ফাংশন তৈরি করো।

এই ফাংশনটি ডেটাবেস থেকে শুধুমাত্র সেই জার্নালগুলোই ফেচ করবে যেগুলোর isActive: true।

কোডটা এমন মডিউলারভাবে লিখবে যাতে ভবিষ্যতে এখানে MongoDB Vector Search বা LangChain সহজেই প্লাগ-ইন করা যায়।
